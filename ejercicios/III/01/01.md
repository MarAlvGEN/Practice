En este ejercicio practicarás cómo dividir un texto en piezas útiles, normalizar datos y construir un resumen estadístico básico usando estructuras clave-valor. Recorrer colecciones con `for...of` y verificar valores con herramientas como `includes()` forma parte natural de este tipo de solución en JavaScript.

---
## Objetivo del Reto

Crea una función llamada `inventarioPalabras(texto)` que reciba un string por parámetro y muestre en consola información útil sobre las palabras que contiene.

## 1. Cuenta cuántas palabras válidas hay

El programa debe separar el texto en palabras usando los espacios como referencia. Si el usuario escribe espacios de más al inicio, al final o entre palabras, no deben contarse como palabras extra.

Por ejemplo, en un texto como `" hola mundo hola "`, el total correcto debe ser `3`.

## 2. Encuentra cuántas palabras únicas existen

Si una palabra aparece varias veces, solo debe contarse una vez dentro del total de palabras únicas.

Por ejemplo, en `"sol luna sol"`, existen `3` palabras totales pero solo `2` palabras únicas.

## 3. Detecta la palabra más repetida

Debes indicar qué palabra aparece más veces y cuántas repeticiones tiene. Para este punto, `"Casa"` y `"casa"` deben considerarse la misma palabra..

Por ejemplo, en `"Perro gato perro PERRO"`, la palabra más repetida es `"perro"` con `3` apariciones.

## 4. Muestra si existe una palabra prohibida

Debes revisar si dentro del texto aparece alguna de estas palabras: `"spam"`, `"fake"` o `"clickbait"`.

Si aparece al menos una, muestra `"Se detectó lenguaje prohibido"`. Si no aparece ninguna, muestra `"Texto limpio"`.

## Ejemplo de Ejecución

```
Escribe una frase:  JavaScript es genial y javascript es decente pero fake 

Total de palabras: 8 
Palabras únicas: 7 
La palabra más repetida es "javascript" con 2 apariciones

Se detectó lenguaje prohibido
```
---

## Reglas del Reto

- Debes crear **obligatoriamente** la función `inventarioPalabras(texto)`.
- Debes ignorar espacios repetidos.
- Debes comparar palabras sin distinguir mayúsculas de minúsculas.
- No uses métodos avanzados como `reduce()` o `map()`.

## Conceptos Clave y Herramientas

Para resolver este reto de manera óptima, se recomienda repasar y utilizar los siguientes recursos:

- **[String.prototype.trim()](https://developer.mozilla.org/es/docs/Web/JavaScript/Reference/Global_Objects/String/trim)** — Para eliminar los espacios en blanco sobrantes al inicio y al final del texto antes de procesarlo.
- **[String.prototype.split()](https://developer.mozilla.org/es/docs/Web/JavaScript/Reference/Global_Objects/String/split)** — Para dividir el string en un array de palabras utilizando los espacios como separador.
- **[String.prototype.toLowerCase()](https://developer.mozilla.org/es/docs/Web/JavaScript/Reference/Global_Objects/String/toLowerCase)** — Para normalizar el texto a minúsculas y garantizar que la comparación no distinga entre "Casa" y "casa".
- **[Array.prototype.includes()](https://developer.mozilla.org/es/docs/Web/JavaScript/Reference/Global_Objects/Array/includes)** — Para validar de forma limpia si las palabras procesadas contienen elementos del array de palabras prohibidas (`['spam', 'fake', 'clickbait']`).
- **[Trabajar con objetos en JavaScript](https://developer.mozilla.org/es/docs/Web/JavaScript/Guide/Working_with_objects)** — Para estructurar un diccionario clave-valor `{}` que guarde el conteo de repeticiones de cada palabra (objeto de frecuencias).
- **[Bucle for...of](https://developer.mozilla.org/es/docs/Web/JavaScript/Reference/Statements/for...of)** — Para iterar o recorrer de forma directa y legible cada palabra dentro del array generado.
